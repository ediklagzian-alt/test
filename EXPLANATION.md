# Vadim-Bank Online Prototype

Прототип банковского приложения, разработанный в соответствии с архитектурными требованиями (SOLID, паттерн Bridge) и логикой из предоставленного задания.

## Основные возможности

1.  **Пошаговая регистрация клиента**:
    *   Шаг 1: Имя и Фамилия (Обязательно).
    *   Шаг 2: Адрес (Опционально).
    *   Шаг 3: Паспорт (Опционально).
2.  **Статус "Сомнительный счет"**:
    *   Если у клиента не указан адрес или паспорт, все его счета помечаются как сомнительные.
    *   Для таких счетов действует лимит на снятие и перевод: **50 000 ₽**.
3.  **Три типа счетов**:
    *   **Дебетовый**: Стандартный счет, нельзя уходить в минус.
    *   **Депозит**: Средства блокируются на 1 год (нельзя снимать/переводить до истечения срока).
    *   **Кредитный**: Можно уходить в минус до 100 000 ₽. Если баланс отрицательный, взимается фиксированная комиссия (500 ₽) при каждой операции.
4.  **Отмена транзакций (Undo)**:
    *   Все операции (пополнение, снятие, перевод) сохраняются в истории.
    *   Любую активную операцию можно отменить одним нажатием, что вернет балансы в исходное состояние.

## Архитектура и паттерны

### 1. Структурный паттерн "Мост" (Bridge)
Разделяет абстракцию управления банковской системой и реализацию интерфейса.
*   **Abstraction**: `BankController` — содержит логику управления (перевод, пополнение и т.д.).
*   **Implementation**: `IBankUI` — интерфейс, который описывает, как UI должен реагировать на изменения.
*   **Concrete Implementations**: 
    *   `WebBankUI` (React-интерфейс).
    *   `ConsoleBankUI` (Консольная заглушка).
*   **Обоснование**: Благодаря этому паттерну мы можем заменить веб-интерфейс на консольный или сетевой (например, через REST API), не меняя код в `BankController` или `BankSystem`. Мы просто подставляем другую реализацию `IBankUI`.

### 2. Принципы SOLID
*   **S (Single Responsibility)**: `Client` хранит данные, `Account` управляет балансом, `Transaction` отвечает за отмену, `BankSystem` — за оркестрацию.
*   **O (Open/Closed)**: Новые типы счетов (например, "Детский" или "Инвестиционный") можно добавить, просто создав новый подкласс `Account`, не меняя `BankSystem`.
*   **D (Dependency Inversion)**: `BankController` зависит от абстракции `IBankUI`, а не от конкретного класса `WebBankUI`.

## Как запустить

1.  **Установка зависимостей**:
    ```bash
    npm install
    ```
2.  **Запуск сервера разработки**:
    ```bash
    npm run dev
    ```
3.  **Сборка проекта**:
    ```bash
    npm run build
    ```

## Использование прототипа

1.  **Регистрация**:
    - Введите имя и фамилию. 
    - Попробуйте пропустить шаг с адресом или паспортом, чтобы увидеть статус **"Сомнительный счет"**.
2.  **Снятие лимитов**:
    - В разделе "Профиль клиента" нажмите кнопку **"Подтвердить личность"**.
    - Заполните недостающие данные (адрес и паспорт) — статус сомнительного счета исчезнет, а лимиты (50 000 ₽) будут сняты.
3.  **Тестирование переводов**:
    - Пополните свой счет на любую сумму (кнопка "Пополнение").
    - В форме перевода нажмите синюю кнопку **"ТЕСТ-ID"** — она подставит номер системного счета `1111111111111111`.
    - Введите сумму и отправьте перевод.
4.  **Управление счетами**:
    - Нажмите `+` над списком счетов, чтобы открыть Депозит или Кредитный счет.
    - При открытии нескольких счетов в форме перевода появится выпадающий список для быстрого перевода между ними.
5.  **Отмена операций (Undo)**:
    - В "Истории операций" нажмите иконку стрелки (Undo) рядом с любой выполненной транзакцией.
    - Система мгновенно вернет средства на исходные счета и пометит транзакцию как отмененную.
